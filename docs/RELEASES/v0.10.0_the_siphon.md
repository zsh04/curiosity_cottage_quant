**Phase 10 Complete.** The system now possesses the capability to ingest live equity market data from Alpaca, specifically targeting the IEX feed. This data is normalized into the **Ezekiel Tick Protocol** and distributed via the Redis Event Bus.

## üöÄ Features

### 1. The Siphon (`app/services/ingest.py`)

- **Real-time Ingestion:** Connects to Alpaca `StockDataStream` (WebSocket).
- **Normalization:** Converts vendor payloads to `market.tick.{symbol}` JSON events.
- **Resilience:** Implements a "Pipeline Repair" loop to auto-recover from connection drops (exponential backoff).
- **Scope:** Configurable `WATCHLIST` (Default: SPY, NVDA, AAPL).

### 2. The Vault (`app/core/config.py`)

- **Expanded Keys:** Added support for:
  - `ALPHAVANTAGE`
  - `FINNHUB`
  - `TWELVEDATA`
  - `MARKETSTACK`
  - `TIINGO`
- **Feed Selection:** Configurable `ALPACA_DATA_FEED` (default: `iex`).

### 3. Documentation

- **Specs:** Added `docs/01_ARCHITECTURE/03_ingest_specs.md` defining the Ingest Service interface and schema.
- **Roadmap:** Updated `docs/02_MISSION/roadmap.md` to reflect completion of Phase 10.
